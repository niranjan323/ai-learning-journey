# 13 — Spec-Driven Development

The capstone workflow: build software by writing clear specifications first, then
letting coding agents implement against them. Combines everything from the journey
into a repeatable, agent-friendly engineering practice.

## 🎯 Learning Objectives

- Write precise, testable specifications for software tasks
- Drive coding agents from specs rather than ad-hoc prompts
- Iterate: spec → implement → verify → refine
- Keep humans in the loop at the right checkpoints
- Combine specs with agentic coding assistants (module 12)

## 🧩 Key Concepts

- **Spec-first workflow** — define *what* before *how*
- **Executable specs** — specs tied to tests and acceptance criteria
- **Agent collaboration** — delegating implementation to coding agents
- **Verification loops** — confirming the implementation meets the spec
